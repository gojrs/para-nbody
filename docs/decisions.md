# Decisions

## D-001 — Reference and Experimental Engines

Status: accepted

`grav-charge` is treated as the traditional/reference engine.

`para-nbody` is treated as the experimental engine.

Rationale:

The project needs a stable comparison target while the new engine evolves.

## D-002 — Frontend Deferred

Status: accepted

Frontend technology is intentionally undefined during early development.

Rationale:

The development cycle should prioritize deterministic physics behavior, API shape, tests, and comparison data.
